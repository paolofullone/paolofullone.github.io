SignalR

Paginated WebAPI vs SignalR

 Posted on April 8, 2025  |  1 minutes  |  126 words  |  Paolo Fullone

This project utilizes SignalR to send real-time data to one or more clients via WebSockets, in this example we are streaming the data from a SQL Server database to the clients.

Instead of making a request/response paginated WebAPI, we decided to use SignalR in this specific project and the Consumer handles the data received. It was applied into a massive data scenario, were we could inquire the database for all orders confirmed by the stock exchange to evaluate some specific scenarios.

Multiple Consumers Challenge

We decided to use Kafka as our message broker to decouple the services and allow for asynchronous communication between them.

This application was responsible to retrieve millions of operations on a daily basis and calculate the commission of each associated office and financial advisors.
