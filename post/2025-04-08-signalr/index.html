Paginated "Web API vs SignalR

 Posted on April 8, 2025  |  1 minutes  |  125 words  |  Paolo Fullone

This project utilizes SignalR to send real-time data to one or more clients via WebSockets. In this example, we are streaming data from a SQL Server database to clients..

Instead of making a request/response paginated “Web API, we decided to use SignalR in this specific project and the Consumer handles the data received. It was applied into a massive data scenario, where we could inquire the database for all orders confirmed by the stock exchange to evaluate some specific scenarios.

In Github there is a detailed explanation of the project and how to run it. The project is built with .NET 8, and the database is SQL Server. The project is also containerized with Docker Compose, so you can run it locally without any hassle.
